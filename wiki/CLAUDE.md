# Wiki Conventions

This `wiki/` folder is an Obsidian vault — a human-readable knowledge graph of project info, decisions, research, design notes, contacts, and daily logs. It is *separate* from:

- `.claude/memory/` — machine-curated memory (auto-injected into AI context)
- `docs/` — formal project docs, PRDs, specifications, public-facing artefacts

If you (Claude) are unsure which of the three belongs, prefer `wiki/` for: half-formed thoughts, decisions and their rationale, things-to-remember, links between concepts, daily progress notes.

## Folder layout

| Folder | Contents | Filename pattern |
|---|---|---|
| `wiki/` (root) | `index.md` only | `index.md` |
| `wiki/daily/` | One file per day, free-form log | `YYYY-MM-DD.md` |
| `wiki/notes/` | Evergreen atomic notes — one concept per file | `kebab-case-title.md` |
| `wiki/people/` | Person profile per file | `firstname-lastname.md` or handle |
| `wiki/projects/` | Sub-project / experiment trackers | `kebab-case-name.md` |
| `wiki/mocs/` | Maps of Content — index notes that link to clusters of related notes | `area.md` (e.g. `game-design.md`) |

## Naming

- All filenames: lowercase, kebab-case, `.md` extension. Example: `chess-narrative-engine.md`.
- Note titles inside the file: H1 in Title Case. Example: `# Chess Narrative Engine`.
- Daily notes: `YYYY-MM-DD.md` (no title needed, date is the title).

## Linking

- Use Obsidian wikilinks: `[[note-name]]`. No `.md` extension in the link.
- Link eagerly. If a concept is mentioned twice, make it a note and link both.
- Linking across folders works without prefix as long as note basenames are unique. If a name collides, use `[[folder/note-name]]`.
- Backlinks are auto-generated by Obsidian — don't maintain them manually.

## Maps of Content (MOCs)

- An MOC is a hand-curated index note. Lives in `wiki/mocs/`.
- Each area gets one MOC: e.g. `mocs/game-design.md`, `mocs/architecture.md`, `mocs/decisions.md`, `mocs/research.md`.
- When you add a new note in `notes/`, add a link to it from the relevant MOC in the same edit.
- Top-level `wiki/index.md` is an MOC of MOCs — links to every MOC and active project page.

## Daily notes

- Created on demand at `wiki/daily/YYYY-MM-DD.md`.
- Free-form. Capture: what happened today, decisions made, open questions, things to revisit.
- When something in a daily note becomes worth keeping long-term, lift it into `notes/` (or update the relevant MOC) and link to it from the daily note.

## Decisions

- Significant decisions go in `notes/` as atomic notes prefixed with `decision-`. Example: `notes/decision-supabase-vs-firebase.md`.
- Each decision note: context, options considered, what was chosen, why, date.
- Linked from `mocs/decisions.md`.

## When you (Claude) should write to the wiki

- User asks to "write this down", "save this", "remember this for later" — and it's not a memory-system fact (preferences, conventions, identity).
- A decision is made — capture as `notes/decision-*.md`.
- User describes a person, contact, collaborator — `people/*.md`.
- New sub-project or experiment kicks off — `projects/*.md`.
- During work that feels notable — append to today's daily note.

## When NOT to use the wiki

- AI-context facts (user role, conventions, tool configs) → `.claude/memory/`
- Formal specs, PRDs, public docs → `docs/`
- Code or code comments → in the code

## Edits and links

When you create a new note, also:
1. Add a wikilink to it from the relevant MOC.
2. If the MOC didn't exist yet, create the MOC and add it to `index.md`.
3. If today's daily note exists, append a one-line reference: `- created [[note-name]]`.
