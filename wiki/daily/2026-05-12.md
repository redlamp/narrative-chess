### /admin surface

Single route, four panels stacked, role-guarded at both the Next.js proxy layer and the server-component layer (defense in depth).

- **Stats** ‚Äî total users (admin / player / bot breakdown), total games (by status), signups + games started in last 7 days, plus recent-signups + recent-games tables. Bots filtered out of recent-signups by default.
- **Users** ‚Äî paginated table with role `<select>` per row (player / admin / bot), self-demotion blocked client + server side. "Hide bots" URL toggle. Muted gray pill on bot rows.
- **Invite codes** ‚Äî generate form (note + 7/30/never expiry), copy-to-clipboard codes, filter chips (all / unused / used / expired), revoke. Code alphabet is Crockford-style base32 (no I/L/O/U) for read-aloud-ability.
- **Danger zone** ‚Äî three nuke buttons with typed-name confirmation (case-sensitive). The non-admin nuke also requires a backup checkbox. Each action writes a row to `admin_audit` before deleting.

Editorial theme reused ‚Äî Fraunces stat numbers, oxblood for destructive UI, parchment cards.

### Signup gate

`/sign-up` form gains an `inviteCode` field at the top (8 chars base32). Server action validates the shape, calls `supabase.auth.signUp()`, then atomically consumes the code via `consume_invite_code(code, user_id)` RPC. On consume failure, the just-created auth user is rolled back via `auth.admin.deleteUser()`. RPC-level errors map to tester-facing strings (`not recognized` / `already used` / `expired`).

On success, redirect to `/check-email?email=...`. AuthDialog's signup variant now routes to `/check-email` instead of `/games` (signup with email-confirm-on returns no session).

Email confirmation flow scaffolded (`/auth/confirm` route handler exchanges the token, `/reset-password` request + `/reset-password/new` set-new-password forms). Supabase dashboard toggle remains a manual step at ship time.

### Gotchas + lessons

#### Next.js 16 renamed middleware.ts ‚Üí proxy.ts

First CI run failed. Both `next dev` (Playwright webServer hang) and `vercel build` (hard fail) error when both files exist. Wrote `middleware.ts`, project already had `proxy.ts`. Fix: merge admin gate logic into existing `proxy.ts`, delete `middleware.ts`. Captured in global memory at `~/.claude/memory/tools/nextjs-16-middleware-to-proxy.md` ‚Äî cross-project gotcha (LLMs trained on pre-Next.js-16 docs default to `middleware.ts`).

#### Service-role env scope on feat-branch previews

Second smoke failure: 500 on `/admin` preview deploy. Cause: `SUPABASE_SERVICE_ROLE_KEY` was scoped to Preview-**dev** branch only; feat branches got nothing. Widened to all preview branches via Vercel dashboard. The decision-auth-email-password.md note updated to reflect this. Closed-beta-OK; reconsider before public release.

#### React 19 purity + server-rendered `Date.now()`

`Date.now()` inside `StatsPanel` (server component) + `InviteCodesPanel`'s `useMemo` both tripped `react-hooks/purity`. Fix per `~/.claude/memory/tools/react-19-lint-patterns.md`: compute `nowMs` once at the top of `page.tsx` (with single `eslint-disable-next-line` since server components render once per request, no ambiguity), pass as `nowMs` prop to both panels.
