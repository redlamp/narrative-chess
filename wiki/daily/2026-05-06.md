## Hydration mismatch fix on Clock + tickNow (PR #28)

Manual smoke on Vercel dev preview surfaced a Next.js console error
when a timed game ran. React reported a hydration mismatch on `Clock`'s
`data-displayed-ms` attribute (and the rendered text). Diff:

```
- data-displayed-ms="86382035"  (server)
+ data-displayed-ms={86381816}  (client)
```

### Root cause

Both `Clock.tsx` and `GameClient.tsx` seeded a clock-tick state via
`useState<number>(() => Date.now())`. The initializer runs once at SSR
(with the server's wall clock) and once at hydrate (with the client's),
producing different values for the first paint. React flags + refuses
to patch.

### Fix

Initialize both states to `null`. First render (SSR + initial hydrate)
uses raw `remainingMs` (Clock) / treats `myClockExpired` as `false`
(GameClient). Mount effect seeds the actual `Date.now()` post-hydrate;
interval keeps it fresh.

- `Clock.tsx`: `now` init → `null`. When `now === null`, `displayed = remainingMs` (no interpolation). Tick interval skips while null.
- `GameClient.tsx`: `tickNow` init → `null`. `myClockExpired` short-circuits to `false` while null. Mount effect seeds `tickNow = Date.now()` even outside `in_progress` so the guard activates immediately post-hydrate.

PR #28 → CI green → merged via `--merge` (`--no-ff`). Merge commit
`99d4890`. Branch deleted local + remote.

### Why this didn't show up earlier

E2E spec runs in headless Chromium against `localhost:3000` dev server.
Dev mode in Next 16 uses Turbopack but the page hydrates the same way.
Possible explanations:

1. The e2e was reading `data-side` and `data-active` attrs but not
   `data-displayed-ms`, so didn't fail on the mismatch (React still
   logs to console but doesn't throw).
2. CI Playwright job is currently no-op (task #13 — `compgen` glob
   bug), so the spec never ran in CI to catch it pre-deploy.
3. Local-dev didn't see it because the user only smoked on Vercel
   preview, where SSR + edge-cached HTML widens the now-deltas.

(2) is fixable; raising priority of task #13.

## Pattern memo

Cross-project takeaway already added to
`~/.claude/memory/tools/react-19-lint-patterns.md` last session.
Updating the same file to add the SSR hydration variant: when a
component's render touches `Date.now()` (directly or via `useState`
initializer), the value diverges between server and client. Pattern
to use:

```tsx
const [now, setNow] = useState<number | null>(null);
useEffect(() => { setNow(Date.now()); /* + interval */ }, [...]);
const displayed = now === null ? fallback : computeFromNow(now);
```

Server and client both render `fallback` on the first frame; client
swaps to the live computation post-hydrate.
