### What landed

- **Move-list polish**
  - CSS-driven entry tween — replaces GSAP useGSAP-after-hydration approach. Fires from first SSR paint, kills the ~2s perceptible delay user reported.
  - Quadratic gap-decay stagger curve: gap(0)=30ms → gap(N-2)=5ms across N cells. Closed-form running sum via the standard sum-of-squares identity, no per-render loop.
  - `baseline` state locks at first paint via useEffect; cells with `cellPos < baseline` keep their original delay forever, fresh arrivals use a different keyframe via `.move-cell-fresh`.
  - Black-cell placeholder dropped during white's turn (no more pre-render of black square before they've moved).
  - Optimistic move-list entry appears at piece-place time (commitOptimisticAndSubmit append), not server-confirm. Dedup-by-ply absorbs the canonical version when it lands.

- **Scrub playback**
  - GSAP timeline walks the chessboard through every intermediate FEN on click. Per-move duration: `clamp(20, 200, 2000/N)` ms; bounded ~2s reveal regardless of game length.
  - `displayFen` overridden by `scrubPlaybackFen` during playback so `viewedPly` updates can lag the board.
  - `viewedPly` walks alongside the timeline so move-list highlight tracks the painted board.
  - Drag disabled (`scrubAnimDuration === null` gate) during playback.
  - Re-click mid-playback: kills tl, takes `currentScrubPlyRef` as new start, builds fresh tl. Auto-snap (livePly bump) preempts: kills tl, snaps to live.

- **Header controls**
  - 4 step buttons: ArrowLeftToLine | ArrowLeft | ArrowRight | ArrowRightToLine (lucide).
  - Play button on header row (top-right of "Move list" title): solid Play icon (`fill="currentColor"`), 1s/move pacing decoupled from piece tween (200ms), `bg-accent text-accent-foreground` when actively playing.
  - `handleScrub(target, opts: { paceMs?, isPlay? })` — opts upgrade from positional `paceMs`. Step buttons + cell clicks omit opts (curve, isPlaying=false). Play passes `{ paceMs: 1000, isPlay: true }`.

- **Piece easing unified**
  - Replaced react-chessboard default `ease` curve (visible asymmetry between fast and slow tweens chained) with `cubic-bezier(0.4, 0, 0.2, 1)` Material standard / chess.com / chessground convention.
  - Applied via `[data-piece] { transition-timing-function: ... !important }` globally so it wins over react-chessboard's inline shorthand without clobbering duration.
  - Default live tween 200ms (was 300ms react-chessboard internal).

- **Captured strip**
  - First icon `marginLeft: -8` to align with player name baseline (SVG transparent margin would otherwise inset visually).
  - Successor overlap `-12px` (was -8) — denser ribbon that still reads piece-by-piece.

### Two Figma plugin gotchas hit (promoted to global memory)

- Bound paint's `opacity` field stores correctly but **renders ignored** — must put alpha at the node level on a child shape that holds the bound fill. → `~/.claude/memory/tools/figma-bound-paint-alpha.md`
- Manual `x` / `y` on auto-layout children is a no-op — need `parent.layoutMode = 'NONE'` AND `child.layoutPositioning = 'ABSOLUTE'` (the latter doesn't auto-clear on disable). → `~/.claude/memory/tools/figma-autolayout-positioning.md`
