Three new lint patterns appeared:

### CLI gotcha ‚Äî `vercel env add ... preview --yes`

Vercel CLI v53.1.0: `vercel env add NAME preview --value V --yes`
errors `git_branch_required` even though docs say omit branch =
"all preview branches". Workaround: pass branch explicitly,
e.g. `vercel env add NAME preview dev --value V --yes`. Pinning
to `dev` is what we want anyway per `domain/auth.md` scoping
pattern.

Also: `vercel env pull --environment=production` is sandbox-blocked
(bulk production env dump). Path of least resistance for replacing
a secret across multiple environments: `rm` the existing entry,
`add` fresh secret to each environment in sequence with `--value`
+ `--yes`.
