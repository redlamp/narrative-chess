### React 19 lint surprises

Three new lint patterns appeared:

- `react-hooks/set-state-in-effect` — `setNow(Date.now())` triggered for re-snap on prop change. Pattern in repo: wrap with `eslint-disable react-hooks/set-state-in-effect` (already used elsewhere in `GameClient.tsx`).
- `react-hooks/purity` — calling `Date.now()` directly in a derived render value (e.g. `myClockExpired`). Fix: tick a `now` state via `setInterval(500ms)`, derive expired from that. No render-time impurity.
