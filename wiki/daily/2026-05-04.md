## WebGL Context Lost — solved (twice)

Round 1: `reactStrictMode: false` in `next.config.ts` (Strict-Mode
double-mount disposes the renderer GL context).

Round 2 (today): hero still flashed-then-blanked even with strict mode
off. Browser console showed `THREE.WebGLRenderer: Context Lost.` after
mount but no auto-restore. Cause: Turbopack HMR re-instantiates the
module + spawns a new `<Canvas>` whose createRoot rebinds GL state. The
browser won't attempt restoration unless the `webglcontextlost` listener
calls `e.preventDefault()`.

Fix: `<Canvas onCreated={attachContextRecovery}>` adds
`webglcontextlost` (preventDefault) + `webglcontextrestored`
(invalidate). Captured both layers in
[[lesson-webgl-strict-mode-context-loss]].

## Process — `--no-ff` for feat → dev

User noticed git history doesn't show branch fan-out anymore — every
feat-branch had been squashed into dev, collapsing the visualization.
Switched policy:

- feat → dev: `gh pr merge --merge` (no-fast-forward, preserves merge
  arc + commits)
- dev → main: stays `--squash` (linear history required by main
  protection)
- never `--rebase` (rewrites SHAs of others' commits)
