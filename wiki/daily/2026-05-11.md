### What landed

#### Frame

- **Layout (tabs).** `Now playing` shelves (active → your open → others' open). `Archive` tab for completed. Limit 40. Tab swap replays the GSAP entrance.
- **Book card composition.** Outer leather cover + inset parchment page. Asymmetric inset (2px top hairline / 6px sides / 28-30px bottom pool) drives the "books resting on leather" read. Padding (not page-margin) creates the leather frame so the bottom 4px-ish is guaranteed by the box model.
- **Cover gradient.** Static top-to-bottom with a saturated `oklch(from var(--book-bg) calc(l + 0.22) calc(c * 1.75) h)` peak band at 16% from top — brighter AND more saturated, not the milky-grey result of mixing toward white.
- **Cursor-followed warm radial highlight** layered on top, 480px radius. Position driven by `--book-light-x / --book-light-y`. **Lighting is global** — one window-level rAF tracker in `<GamesLibrary>` writes the cursor's relative position into every visible `.book-cover` per frame, so one cursor lights every book on the shelf simultaneously.

#### Outcome-coloured covers

| State | Cover | Token |
|---|---|---|
| `in_progress` / `open` | navy | `#1e3a55` |
| Viewer won (decisive) | forest | `#2f4a32` |
| Viewer lost / observer-of-decisive | oxblood | `#5c1820` |
| `draw` / `aborted` | warm slate | `#4a4943` |

Eyebrow ink mirrors the cover hue via `oklch(from var(--book-bg) calc(l + 0.14) calc(c * 1.35) h)`; dark mode lightens harder (`l + 0.36`) so the text reads against the warm-taupe page. "Your move" stays on `var(--oxblood)` as signal.

#### Floating cursor preview

`<CursorPreview />`, mounted once at the library root. Three layered transform refs:
1. `posRef` — `translate(cursorX, cursorY)`, instant, follows pointer.
2. `offsetRef` — `translate(±x, ±y)` for quadrant, with a 320ms CSS transition so the panel slides between corners when flipping near a viewport edge.
3. `tiltRef` — `perspective(700px) rotateX/Y` driven by normalised cursor screen position, signed-sqrt curve, ±7° / ±2.5°.

Hover context provides `setActive(card)` / debounced `clear()` (220ms) so cards in adjacent grid cells never let the preview blink off in the gap between them. `setActive` dedupes by card id to skip re-renders on intra-card mouse motion.

`<AnimatedBoard>` inside the preview tracks pieces across FEN changes. Each (color, type) cohort greedy-matches by nearest neighbour, so a knight that's on b1 in game A tweens b1 → c3 when hovering over game B. Captured pieces flank the board on their *own* starting side (white captures stack at the bottom strip, black at the top) so tween distance off the board is minimal. Orientation pinned to white-bottom regardless of viewer side — flipping mid-hover would teleport every piece across.

#### Mobile + touch

- `@media (hover: hover)` guards the scale + lifted shadow so a tap doesn't stick the hover state.
- Touch can't trigger the cursor preview (no `mouseenter`). Each non-open-invitation card renders a tiny static board (`<StaticBoard size={14} />` → 112×112) anchored to the top-right of the page. `max-md:pr-[128px]` on the inner content wrapper + `min-h-[120px]` keep the HR settling below the bottom of the board.
- HR spans the full page width (right gutter is reserved on the inner content wrapper only, not on the page itself).

#### Page layer + dark mode

- New tokens `--book-page-top / --book-page-bot`. Light: parchment (existing palette). Dark: warm taupe `#3d3830 / #2e2a23` — slightly lighter than `--background` so the page reads as a lit surface on the leather rather than disappearing into ambient dark.
- Dark mode also layers two faint repeating gradients on the page background as aged-paper grain so it doesn't look like a flat dark rectangle.
- After GSAP entrance, `clearProps: "transform"` releases inline transform control so CSS `:hover` (scale 1.04 from centre, translateY -2px, lifted shadow) actually takes — without this, the inline `transform: translate3d(0,0,0) rotate(0)` GSAP leaves behind blocks the hover scale silently.

### Cross-cutting wins (promoted to global memory)

- **GSAP `clearProps` after entrance** — `gsap.fromTo` writes the final `transform` as an inline style; CSS `:hover { transform }` silently fails because inline beats selector specificity. Always `onComplete: () => gsap.set(target, { clearProps: 'transform' })` if you want CSS hover to take over after the entrance. Promoted to `~/.claude/memory/tools/gsap-clearprops-after-entrance.md`.
- **`oklch(from var(--token) calc(l + N) calc(c * M) h)`** — relative color syntax for lightening + saturating that respects the source's hue. Replaces `color-mix(..., white)` which adds neutral and desaturates. Requires modern browsers (CC119+ / Safari 16.4+) and `@property` declarations to animate custom-prop transitions. Promoted to `~/.claude/memory/tools/css-relative-oklch-and-property.md`.
- **`@media (hover: hover)`** — gate `:hover` CSS so a tap on a touch device doesn't leave the hover state stuck. Pair with `@media (pointer: fine)` for mouse-specific behavior. Promoted to `~/.claude/memory/tools/media-hover-pointer-touch-safety.md`.

### Notes

- Iteration count on this pass was high — book metaphor heaviness, frame thickness, gradient peak brightness, eyebrow color, tilt magnitude, board size, hover scale visibility, mobile board geometry, footer column layout, light source globality, page top padding, archive piece icons all went through multiple rounds of feedback. The path was real design exploration rather than over-engineering.
- The "global cursor lighting tracker writing CSS vars on every cover per frame" was the most satisfying mechanic discovered this session. Reads as one consistent light source moving across the shelf. Cheap (one window listener, rAF-throttled).
- The cross-FEN piece-tween was the second — pieces find their nearest counterpart in the next game's position and tween there, captured pieces flow to a strip on their starting side. The library starts to feel alive when you sweep the cursor.
