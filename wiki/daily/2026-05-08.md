# 2026-05-08

## Three.js + oklch fill bug (commit 00cb914)

Tested PR #35 Vercel preview. Floor color did not respond to theme
toggle in dark mode. The "Two players sit..." deck text became
unreadable against the still-cream floor.

### Root cause

Scene tokens (`--scene-floor`, `--plinth-color`, `--piece-light`,
`--piece-dark`, `--scene-fog`) were defined as `oklch(...)` strings.
**Three.js material `color` does not parse oklch**, only hex / rgb
/ hsl / named colors. The MutationObserver on `documentElement.class`
fired correctly on theme change, the new oklch string reached the
material, and Three.js silently fell back to default white-ish.

### Fix

Convert Three-consumed tokens to hex literals in `globals.css`:

| Token | Light | Dark |
|---|---|---|
| `--scene-floor` | `#ddcca0` | `#1f1812` (deeper for cream-text contrast) |
| `--scene-fog` | `#ebe2d2` | `#181210` |
| `--plinth-color` | `#5a4128` | `#3a2a1a` |
| `--piece-light` | `#f0e7cf` | `#dfcca0` |
| `--piece-dark` | `#332618` | `#0a0805` |

`--scene-bg-top` / `--scene-bg-bot` stay oklch (CSS-only gradient,
never reaches Three.js).

Promoted to global memory as `tools/r3f-strict-mode-context-loss.md`
neighbor, but distinct: this is the **CSS-token-to-Three-color**
bridge, not the StrictMode context loss. Worth its own note if
encountered again.

### Procedural pieces -> Cburnett SVGs

First pass built piece silhouettes from primitives (ellipses, rectangles,
vector paths). Result looked off, especially the knight horse-head
approximation. User pointed out the live game uses SVG pieces.

`react-chessboard` v4.7 ships the **Cburnett chess set** (Wikipedia /
lichess, CC BY-SA 3.0). Downloaded 12 SVGs from
`raw.githubusercontent.com/lichess-org/lila/master/public/piece/cburnett/`.
Replaced procedural shapes inside each Piece variant via
`figma.createNodeFromSvg(svgString)` + `rescale(200/45)`. Component
IDs preserved, so all 32 instances on the Game page board updated
automatically.
