# Taylor Piece Set

Default chess piece set for Narrative Chess V2. **Hand-drawn by Taylor** (project owner / artist) — not licensed from a third party, not generated by AI, not pulled from a public set.

## Versions

- **v1** (2026-05-09) — initial set with outlines, drawn in Figma. Twelve SVGs (`wp wn wb wr wq wk bp bn bb br bq bk`) imported into `public/pieces/taylor/`.
- **v2** (2026-05-09) — outlines removed for a flatter, editorial silhouette. Same twelve files, refreshed in place from the same Figma node.

Light pieces use a warm cream-grey gradient; dark pieces use a deep walnut. Both palettes are baked into the SVG (no `currentColor` substitution), so they read consistently on either theme without a runtime swap.

## Source

- Figma file `ow2paLzok6bnWqwHiHDBuP` (Narrative Chess design system), page `04 - Chess Pieces`, set "Taylor".
- Refreshed exports drop into `public/pieces/taylor/` directly. Filenames match react-chessboard's `Piece` keys (`wp`, `wn`, etc.) lowercased.

## Renderer wiring

Mounted via `lib/chess/piece-set.ts` (`taylorPieces: CustomPieces`) and passed to `<Chessboard customPieces={taylorPieces} />`. The renderer is plain `<img src="/pieces/taylor/{file}.svg" />` — `next/image` would be heavier here (next/image disables remote SVG by default and adds wrapper overhead with no caching win for small static assets that re-render every drag tick).

## Why this matters

- **Authorship.** The pieces are original work by Taylor and are part of the project's brand identity, not interchangeable assets. Future "swap pieces" prompts should preserve this set as the project default and route alternates to a user-pickable theme rather than replacing Taylor.
- **Refresh path.** When the Figma source updates, re-export the twelve SVGs from page `04 - Chess Pieces` over the existing files in `public/pieces/taylor/`. No code change required as long as filenames stay stable.
